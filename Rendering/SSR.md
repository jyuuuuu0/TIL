# SSR (Server Side Rendering)

SSR은 서버에서 HTML을 생성하여 브라우저에 전달하는 방식이다.

브라우저는 이미 완성된 HTML을 받기 때문에 초기 화면이 빠르게 표시된다.

대표적으로 Next.js, Nuxt.js 같은 프레임워크에서 사용된다.

## 동작 과정

1. 브라우저가 서버에 페이지 요청
2. 서버가 데이터를 가져와 HTML 생성
3. 완성된 HTML을 브라우저에 전달
4. 이후 JavaScript가 로드되며 Hydration 진행

## 특징

### 장점

- 첫 페이지 로딩 속도가 빠르다.
- 검색엔진최적화가 가능하다.

### 단점

- 초기 로딩 이후 페이지 이동 시 속도가 CSR에 비해 느리다.
- 깜빡임 이슈 (매번 새로고침 해야하기 떄문에)
- 서버 과부화
