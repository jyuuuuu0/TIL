# MPA (Multi Page Application)

여러 페이지로 구성된 전통적인 웹 어플리케이션의 형태로, 사용자의 요청마다 새로운 페이지를 서버에서 벋어 브라우저에 표시하는 방식이다.

## 작동방식

MPA는 사용자가 링크를 클릭하거나 특정 동작을 할 때마다 서버로부터 새로운 HTML 페이지를 받아 전체 페이지를 다시 렌더링 한다.
서버는 요청에 따라 브라우저에서 렌더링할 완전한 HTML 문서를 생성하여 전달한다.

## MPA의 장점

### SEO 친화적

MPA는 검색 엔진이 페이지를 인덱싱하는 방식과 자연스럽게 일치하여 SEO에 더 유리하며, 각 페이지를 개별적으로 최적화할 수 있다.

### 확장성

복잡한 기능과 다양한 유형의 컨텐츠를 처리하기에 적합하다.
각 페이지가 독립적으로 구성되어 있어 기능별로 분리 및 관리가 용이하다.

## MPA의 단점

### 느린 페이지 전환 속도

페이지 이동 시마다 서버와의 통신이 발생하고 전체 페이지를 다시 로드해야 하기 때문에 사용자 경험이 저하될 수 있다.
특히 네트워크 환경이 느린 경우 로딩 지연이 발생할 수 있다.
또한 페이지 전환 시 흰 화면이 잠깐 보일 수 있다.

### 자원 집약접

각 페이지 이동 시 HTML, CSS, JS 등의 리소스를 다시 로드할 가능성이 있어 비효율적일 수 있다.

## MPA 사용 적합한 상황

- SEO가 중요한 서비스
- 각 페이지가 독립적인 콘텐츠를 가지는 경우
  예:
- 쇼핑몰
- 뉴스 사이트
- 블로그
  또한 구조가 단순하고 표준 웹 방식이기 떄문에 구형 브라우저에서도 안정적으로 동작한다.
