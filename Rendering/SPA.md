# SPA (Single Page Application)

SPA는 단일 HTMl 페이지를 로드한 후, 사용자와의 상호작용에 따라 필요한 데이터만 동적으로 가져와 페이지를 새로고침 없이 갱신하는 애플리케이션 방식이다.

## 작동 방식

초기 로딩 시 필요한 HTML, CSS, JS를 한 번에 불러온다.
이후에는 JS를 이용해 사용자 상호작용에 따라 DOM을 동적으로 업데이트한다.

추가 데이터가 필요한 경우 서버와 통신하여 데이터를 받아오고,
페이지 전체를 새로고침하지 않고 화면 일부만 갱신한다.

SPA는 주로 CSR 방식으로 구현된다.
대표적으로 React, Vue.js등이 있다.

## SPA의 장점

### 속도 및 반응성

초기 로딩 이후에는 필요한 데이터만 서버에서 받아오기 때문에 페이지 전환이 빠르고 반응성이 뛰어나다.

### 사용자 경험

페이지 새로고침 없이 화면이 부드럽게 전환되어 네이티브 애플리케이션과 유사한 사용자 경험을 제공한다.

### 자원 효율성

필요한 데이터만 요청하므로 불필요한 리소스 로딩이 줄어들고, 일반적으로 네트워크 사용량이 감소한다.

## SPA의 단점

### SEO 문제

SPA는 초기 HTML에 콘텐츠가 거의 없고 JS 실행 후에야 화면이 완성되기 때문에 검색 엔진이 페이지 내용을 파악하는 데 불리할 수 있다.

최근 검색 엔지은 JS 렌더링을 지원하지만 여전히 SSR에 비해 불리한 경우가 많다.

### 초기 로드 시간

초기 진입 시 필요한 JS 파일의 크기가 크기 때문에 첫 로딩 속도가 느릴 수 있다.

### 브라우저 히스토리 관리

SPA는 페이지 전환이 실제로 일어나지 않기 때문에 브라우저 히스토리를 직접 관리해야 한다.

## SPA 사용 적합한 상황

- 사용자 상호작용이 많은 서비스
- 빠른 UI 반응성이 중요한 경우
  예)
- 이메일 서비스
- 소셜 네트워크
- 협업 도구
