# CSR (Client Side Rendering)

CSR은 브라우저(클라이언트)가 JavaScript를 실행하여 화면을 렌더링하는 방식이다.

초기에는 서버에서 기본 HTML 파일과 JavaScript 파일만 전달하고, 이후 JavaScript가 데이터를 가져와 화면을 구성한다.

대표적으로 React, Vue.js 같은 SPA 프레임워크에서 많이 사용된다.

> SPA 프레임워크란?
> 한 개의 페이지에서 동적으로 화면을 바꿔주는 웹/앱을 만들기 위한 도구.

## 동작 과정

1. 브라우저가 서버에 페이지 요청
2. 서버는 기본 HTML과 JavaScript 파일 전달
3. 브라우저가 JavaScript 실행
4. JavaScript가 API로 데이터를 가져와 화면 렌더링

## 특징

### 장점

- 새로고침이 발생하지 않아 사용자 경험애 도움을 줌.
- 초기 로딩 이후 빠른 웹사이트 렌더링 가능
- 필요한, 변경된 데이터만 받아오므로 트래픽 감소

### 단점

- 검색엔진최적화(SEO) 보완 필요
- 초기 로딩 느림
